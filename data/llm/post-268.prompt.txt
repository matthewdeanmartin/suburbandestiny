If you did have a WAMP (or LAMP) application you wanted to run, it would make more sense to put it on your web host instead of a thumb drive. But since we can, let’s try to rationalize it.\n\n\n**Advantages**Localhost is faster.  \n\nLocalhost is more private.  \n\nLocalhost is disconnected.  \n\nLocalhost doesn’t cancel your account when you go\\-over your download limit.  \n\nLocalhost is free. I’d rather have those on a connected server. If data loss didn’t worry you, then you might consider [time trackers](http://freshmeat.net/projects/timesheet.php/), address databases, etc. These PIM apps were never very compelling before web 2\\.0, now they are much more compelling as multi\\-user social applications.\n\n\nSpeaking of folder browsing, most web\\-based file browsing system suck, but if one existed that didn’t suck, it would be nice to have that on portable XAMPP. I still don’t know of a good portable source control server, but you can use RapidSVN as a portable client.
