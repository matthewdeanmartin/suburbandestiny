This is the most expensive feature. It achieves version resilience by adding 5 assemblies between your host and plug ins. Without code generation, the extra five assemblies adds too much work. \n\n\nAlso, when it comes time to write the V2 to V1 adapters, you will have to make ugly choices like, should the adapters throw errors or should they similate the new functionality. Now how is a component backwards compatible if it throws runtime errors? Officially, you aren’t supposed to pass winforms controls across MAF contracts, ie only the host has UI. This creates a blended version V1/V2 website. Microsoft office is a COM application and as such will work best with a component technology that has a lot of parallels with COM.
