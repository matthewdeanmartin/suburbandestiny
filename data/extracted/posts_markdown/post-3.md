Decision 1\. I’m going to develop this website in the open. I’m a spare time programmer, so I’m not going to have a test/production set up.


D2\. The site will support these transactions:


Create user/update user profile.  

Create/Updated/Cancel canned event.  

RSVP Yes/No to canned event.


D3\. The canned events will all be simple and target a few general special interests that have a social component. Write in. Hike. Dinner. TV Party. Game night.


D4\. No membership fee.


D5\. Use case: People sign up and create a canned event, then they RSVP. Everyone gets the copy of the current calendar in their email box. People browse the profiles to see if they want to go.


Issues to be resolved. Venue managment? How to either use or avoid competing with evite.com \& match.com?
